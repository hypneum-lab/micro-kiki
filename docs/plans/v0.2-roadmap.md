# v0.2 roadmap — placeholder

Not a plan yet. Just a tracker so the intent survives v0.1 shipping.

## Scope

- **Temporal context**: inject real-time clock, location, relevant news slice, and user calendar into the system prompt. Cheap, high leverage, no new training.
- **Future-reasoner**: CoT temporal chains ("if X happened at T, expect Y by T+k"), calendar-aware planning, scheduled-task reasoning. Implemented as a tools layer (external calculator + calendar + retrieval), not a new MoE-LoRA stack.

## Why not a stack

Per arxiv 2601.10132, 4B-class LLMs underperform dedicated time-series ML on quantitative forecasting. A MoE-LoRA stack fine-tuned on temporal Q&A would inherit the base's weakness and add rank-16 adapters on top — same ceiling, more parameters. Context injection + tools is the right layer.

## Prerequisites

- v0.1 shipped (32 stacks + router + cognitive layer + serving + release).
- Aeon memory palace online (temporal recall already covers most "what happened when" questions).
- Tools framework selected (likely MCP-style, reusing patterns from Factory 4 Life).
