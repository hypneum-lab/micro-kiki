Quelle taille d'alphabet pour les neurolettres ?

Rappel du design en cours : nerf bidirectionnel · π↓/ε↑ sémantique · rythmes γ/θ · pool hétérogène MLP+LIF. Le choix de combien de symboles dans l'alphabet partagé entre WMLs fixe l'expressivité, la difficulté d'entraînement, et l'interprétabilité.

A

16 codes — « ADN-like », minimaliste

Information : 4 bits par neurolettre. Comme les nucléotides (A,T,G,C) mais × 4. Signification émerge par séquence longue, pas par symbole unique.

Alphabet = { α β γ δ ε ζ η θ ι κ λ μ ν ξ ο π }  |Σ|=16
Message typique = [γ α π ε ε ξ λ μ … ]    (20-200 lettres)
Très compressé · VQ facile à entraîner · compositionnel fort

Atout : alphabet facile à stabiliser (peu de collapse modes), chaque code largement utilisé. Limite : bande passante faible — besoin de nerfs qui tirent beaucoup pour dire quelque chose.

B

64 codes — « codon-like », compromis

Information : 6 bits par neurolettre. 64 = 4³ (analogie directe codons ARN). Point d'équilibre entre expressivité et stabilité.

Alphabet = { 64 codes, ~6-8 « familles » apprises }
Message typique = [ π₁₇ π₄₁ ε₂₃ π₅ … ]    (8-50 lettres)
Assez pour encoder 1 « concept » par lettre · ~64 clusters

Atout : bonne lisibilité (64 = ~7 phonèmes-like). Entraînement stable avec commitment loss VQ-VAE. Limite : peut saturer si un WML a + que 64 concepts à exprimer.

C

256 codes — « byte-like », expressif

Information : 8 bits par neurolettre. Un octet. Riche pour capturer nuance dans une seule lettre, mais risque de collapse/dead codes si pas entraîné soigneusement.

Alphabet = { 256 codes · fréquences très déséquilibrées }
Message typique = [ π₁₂₇ ε₄₁ π₁₉₉ … ]    (4-20 lettres)
1 concept riche par lettre · collapse risque

Atout : bande passante max. Bien pour WMLs qui gèrent beaucoup de contextes. Limite : dead codes classiques en VQ-VAE, exige EMA codebook update ou rotation tricks.
