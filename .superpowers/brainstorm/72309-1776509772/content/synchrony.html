Quel régime temporel pour les échanges inter-WML ?

Bidirectionnel + alphabet partagé → il faut absolument un mécanisme pour répondre à : « qui parle quand ? ». Trois familles biologiquement plausibles, avec des implications d'implémentation très différentes.

A

Lockstep — Tours discrets synchronisés

Analogie : horloge globale · tous les WMLs émettent puis écoutent, par rounds. Comme les couches d'un Transformer.

t=0 : tous émettent  ━━►  t=1 : tous reçoivent  ━━►  t=2 : …
WMLA ━━[α₂]━━► WMLB       (round 0)
WMLB ━━[δ₃]━━► WMLA       (round 1)
Pas d'ambiguïté · parité impair/pair désambiguïse

Atout : trivial à entraîner (BP standard), débuggable, déterministe. Limite : pas biologique, latence = max des WMLs lents, rythme global rigide.

B

Event-driven asynchrone — Tir spontané

Analogie : neurones biologiques réels — un WML émet quand il a quelque chose à dire (un seuil interne dépassé). Les autres écoutent en temps réel.

t ──────────────────────────────────────────►
WMLA    │α₂│          │γ₇│              │ε₃│
WMLB           │δ₃│                      │β₁₁│
Chaque message = tuple (src, dst, lettre, timestamp)

Atout : biologique, sparse par nature (pas de tir = pas de calcul), énergie optimale. Limite : dur à entraîner (événements non-différentiables → RL / surrogate gradient), ordonnancement non-déterministe.

C

Phase-clocked — Rythmes θ / γ imbriqués

Analogie : oscillations corticales · chaque WML est assigné à une phase (θ 4-8 Hz porte les « items », γ 30-100 Hz porte les « traits »). Un WML n'écoute que ce qui est émis pendant sa phase d'entrée.

phase φ=0 : WMLA,C,F émettent · autres écoutent
phase φ=1 : WMLB,D,E émettent · autres écoutent
phase φ=2 : retour φ=0 …
Multiplexing temporel · alphabet partagé · pas de collision

Atout : biologiquement plausible, déterministe au sein d'une phase, différentiable, élégant pour alphabet partagé. Limite : nécessite un scheduler global des phases (mais apprenable).

Les 2 questions restantes (après celle-ci) : granularité d'un WML (combien de neurones ? un bloc Transformer ? un LoRA ?) et taille de l'alphabet (16 · 64 · 256 · appris). Puis je propose 2-3 approches architecturales avant spec.
